Hivemind is a semantic code indexer and search tool. It indexes local codebases and web documentation, stores vector embeddings in Qdrant, and exposes search capabilities via MCP (Model Context Protocol) and REST API.

== Getting Started

=== Prerequisites

* Python 3.12+
* `uv` package manager (install via `curl -LsSf https://astral.sh/uv/install.sh | sh`)
* A running Qdrant instance (see link:deployment.adoc#docker[Docker setup])
* An OpenAI-compatible embedding API (e.g., LM Studio)

=== Quick Install

[source,bash]
----
# Clone and install
git clone <repo>
cd hivemind
./setup.sh

# Initialize a project
cd /path/to/your/codebase
hivemind init

# Start indexing
hivemind indexer start .

# Search
hivemind search "authentication logic"
----

=== MCP Integration

Configure in your IDE (Cursor, Claude Desktop, etc.):

[source,json]
----
{
  "mcpServers": {
    "hivemind": {
      "command": "/full/path/to/uv",
      "args": [
        "run",
        "--directory",
        "/full/path/to/hivemind",
        "python",
        "main.py",
        "mcp"
      ]
    }
  }
}
----

=== Component Boundaries

The project enforces strict separation of concerns (see link:../AGENTS.md[AGENTS.md]):

* `core/` — The **only** place where external API clients (Qdrant, OpenAI) are instantiated.
* `indexer/` — Handles file reading, chunking, and ingestion. Never handles user queries.
* `server/` — Handles MCP tool definitions and LLM interactions. Never reads local files directly (delegates to `core/filesystem.py`).
* `scout/` — Handles web crawling. Used by the server for `scout_urls` and `deep_research` tools.
* `cli/` — Contains execution scripts and CLI wrappers.
* `infra/` — Contains deployment artifacts only.

== Key Concepts

=== Semantic Search Pipeline

. **Indexing**: Files are discovered → preprocessed → chunked → embedded → stored in Qdrant with flat, filterable metadata.
. **Search**: Queries are embedded → Qdrant performs vector similarity search → results are formatted as Markdown or JSON.
. **Incremental**: Only changed files are re-indexed, tracked via SHA-256 checksums and modification times.

=== Chunking Strategies

| Strategy | Method | Best For |
|----------|--------|----------|
| `ast` (default) | Tree-sitter AST, preserves function/class boundaries | Code files with known languages |
| `by_size` | Fixed character count with overlap | Any text, language-agnostic |
| `by_lines` | Fixed line count with overlap | Uniform line-based analysis |
| Markdown | Header-aware splitting | `.md` files, documentation |

=== Map-Reduce for Web Pages

The scout module implements a Map-Reduce pattern for large web pages:

. **Map**: Crawl URL → chunk content → cache in memory.
. **Reduce (TOC)**: Return Table of Contents with section names and token estimates.
. **Reduce (Sections)**: Return only requested sections from cache.

This avoids context-window overflow from giant documentation pages or GitHub PRs.

== Security Disclaimer

**Hivemind is not hardened against prompt injection.** The MCP server executes tools based on instructions from AI agents, which may be influenced by untrusted content (e.g., web pages crawled via `scout_urls` or `deep_research`, or code files indexed from a repository). An attacker who can inject malicious instructions into crawled content could potentially trick the agent into performing unintended operations.
